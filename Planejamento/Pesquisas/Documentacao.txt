 A documentação de API's é um conjunto de informações qeu explica como usar uma API, seus recursos e como
interagir com ela, Ela é basicamente um guia para desenvolvedores que ajuda a construir interações com a API

*DEVE INCLUIR:*
- exemplos de codigo
- tutoriais
- detalhes sobre funções, classes e tipos de retorno
- explicação dos endpoints
- interpretação do proque de usar os endpoints
- exemplo de como usar os endpoints

Normalmente fica disponivel e hospedada no site principal da empresa ou portal voltado para a API, 
sempre em area especialmente dedicada para este fim.

Ferramenteas como Stoplight, Swagger, SwaggerHub, OpenAPI podem ser utilizadas para a documentação de API's

As Boas Práticas discorrem sobre api da seguinte forma:
- dar grande importancia ao design e formatação
- destacar os principais caminhos terminais
- apresentar informações de maneira concisa e legivel
- maximixar o espaço na tela
... entre outros.
